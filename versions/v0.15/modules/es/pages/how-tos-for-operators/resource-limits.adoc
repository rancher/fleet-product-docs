[#_default-resources]

  --set resources.requests.memory=768Mi

[#_per-component-overrides]
== Sobrescrituras por componente

Los contenedores de controladores individuales pueden sobrescribir el valor predeterminado con configuraciones especﾃｭficas del componente.
Las claves de componente admitidas son:

* `fleetController` : el controlador principal de Fleet
* `fleetCleanup` : el controlador de limpieza
* `fleetAgentmanagement` : el controlador de gestiﾃｳn de agentes
* `gitjob` : el controlador de GitJob
* `helmops` : el controlador de HelmOps

Si una clave de componente estﾃ｡ presente, su valor tiene prioridad sobre el predeterminado `resources.limits` y `resources.requests`. 

Para eliminar las restricciones de recursos para un contenedor independientemente de la configuraciﾃｳn predeterminada, establece un componente como un objeto vacﾃｭo `{}`.

Ejemplo `values.yaml` utilizando sobreescrituras por componente:

[source,yaml]
----

  limits:
    cpu: 4000m
    memory: 4Gi
  requests:
    cpu: 100m
    memory: 256Mi
  fleetController:
    limits:
      cpu: 8000m
      memory: 8Gi
    requests:
      cpu: 250m
      memory: 768Mi
  gitjob: {}         # no resource constraints for gitjob
  helmops: {}        # no resource constraints for helmops
----

En este ejemplo:

* `fleetController` utiliza sus propios lﾃｭmites y solicitudes.
* `gitjob` y `helmops` no tienen restricciones de recursos aplicadas.
* `fleetCleanup` y `fleetAgentmanagement` vuelven al valor predeterminado de nivel superior.

Aplica la configuraciﾃｳn con Helm:

[source,bash]
----

  -f values.yaml
----

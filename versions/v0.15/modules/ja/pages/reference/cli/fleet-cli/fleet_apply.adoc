= Fleetを適用する
:revdate: 2026-02-05
:page-revdate: {revdate}

ディレクトリからバンドルを作成し、それらを出力するか、クラスターに適用します

----
fleet apply [flags] BUNDLE_NAME PATH...
----

=== 知っておくと良いこと

`fleet apply`は環境変数から`known_hosts``データを読み取ります（`FLEET_KNOWN_HOSTS`）、これはgitクローンが行うことに似ています。

これは、`fleet apply`をホストマシン上で直接実行したい場合、`KNOWN_HOSTS`のカスタム値を提供するために、事前に`FLEET_KNOWN_HOSTS`をエクスポートするか、`FLEET_KNOWN_HOSTS=<your_entries> fleet apply <args>`として実行する必要があることを意味します。

同時にバンドルを作成するルーチンの数は、`--bundle-creation-max-concurrency`フラグまたは`FLEET_BUNDLE_CREATION_MAX_CONCURRENCY`環境変数を介して設定できます。デフォルト値は4です。これは、多くのバンドルを含む大規模なリポジトリで作業する際に便利で、バンドル作成の並列性を制御できるようにします。次に例を示します。

`FLEET_BUNDLE_CREATION_MAX_CONCURRENCY=8 fleet apply <args>`
